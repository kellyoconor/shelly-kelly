# Memory Management Rules

## What Gets Written to MEMORY.md

### ✅ PERMANENT (Core Memory)
- **Kelly's core identity** — principles, words, mantras (rarely changes)
- **Shelly's operating principles** — how I work best with Kelly
- **Critical automation** — must-run jobs Kelly depends on
- **Current major context** — active trips, interviews, relationships (~3 month window)
- **Hard-learned lessons** — mistakes that cost time/frustration if repeated
- **Banned phrases/behaviors** — things that upset Kelly

### 📝 TEMPORARY (Working Memory)
- **Recent lessons** — keep only last 5, archive rest
- **Automation notes** — current setup, not historical changes  
- **Context details** — enough for continuity, not full backstory

### 🗄️ ARCHIVE (Kelly's Vault)
- **Detailed lesson history** — full context of what went wrong and how it was fixed
- **Historical automation changes** — what was tried, what didn't work
- **Completed projects/events** — Scotland trip details, finished marathons
- **Old versions** — full MEMORY.md backups with timestamps

## When to Archive

### Triggers for Moving Things Out:
- **Age:** Context >6 months old (unless still actively relevant)
- **Size:** Any section >1,000 characters gets reviewed for archiving
- **Completion:** Finished projects/events get summarized, details archived
- **Redundancy:** Multiple entries about the same thing get consolidated
- **Replacement:** When new lessons supersede old ones

### Monthly Review Process:
1. **Audit current size** — target: <2,500 chars total
2. **Archive old lessons** — keep only last 3-5 major ones
3. **Update current context** — remove completed items, add new ones
4. **Consolidate similar items** — group related automation notes
5. **Backup full version** — before making changes

## Writing Guidelines

### Real-Time Updates:
- **Add immediately:** New critical lessons, banned behaviors, automation changes
- **Update in place:** Current context, contact info, ongoing situations
- **Check length after:** If >3,000 chars, trigger archive review

### Weekly Sweep:
- **Review daily files** — extract 1-2 key insights max
- **Archive old sections** — anything >6 months or >1,000 chars
- **Update current context** — refresh status on ongoing situations
- **Length check** — target staying under 2,500 chars

## Emergency Procedures

### If MEMORY.md Bloats (>4,000 chars):
1. **Immediate backup** to vault
2. **Archive entire "Recent Lessons"** section
3. **Trim current context** to just active items
4. **Consolidate automation notes** to essentials only
5. **Add warning** to weekly sweep job to monitor size

### Length Targets:
- **Target:** 2,000-2,500 characters
- **Warning:** 3,000+ characters (archive review needed)
- **Critical:** 4,000+ characters (immediate action required)

## Implementation

### Automated Monitoring:
- Weekly sweep checks length and suggests archives
- Monthly audit runs full analysis
- Real-time warnings if adding large sections

### Manual Tools:
- `memory-manager.py audit` — analyze current state
- `memory-manager.py optimize` — apply rules automatically
- Archive templates in vault for consistent organization
